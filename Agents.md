# 프로젝트 작업 규칙 (Agents.md)

## 1. 멀티모듈 작업 규칙

- 작업 전 **어느 모듈**인지 명확히 확인
- 모듈별 책임:
  - `kotlin-basic`: Kotlin 문법 학습, 은행계좌 시스템
  - `kotlin-chatbot`: Spring AI, RAG 챗봇

## 2. 실행/빌드 명령어

```bash
# 특정 모듈 실행
./gradlew :kotlin-basic:run
./gradlew :kotlin-chatbot:run

# 전체 빌드
./gradlew build

# 특정 모듈 테스트
./gradlew :kotlin-basic:test
```

## 3. 요구사항 명확화 규칙

**요구사항이 모호하거나 불명확한 경우 반드시 질문하여 구체화**

- **즉시 추측하지 말고 질문한다**
- 확인할 내용:
  - 구체적인 목표와 기대 결과
  - 대상 파일/범위
  - 원하는 결과물의 형태
  - 제약 조건 및 선호 방식
- **AskUserQuestion 도구 적극 활용**
- 답변 받은 후에 작업 시작

## 4. 코드 검증 규칙

**모든 새로 작성된 코드는 반드시 테스트 후 성공 시에만 제공**

- 작성한 코드는 즉시 컴파일/실행 테스트
- 테스트 실패 시 원인 파악 및 수정
- 성공 확인 후에만 사용자에게 전달
- 테스트 방법:
  ```bash
  ./gradlew :kotlin-basic:test
  ./gradlew :kotlin-chatbot:test
  ./gradlew build
  ```

## 5. 정보 재검증 규칙

**사용자에게 전달되는 모든 정보는 반드시 재검증**

- **파일 경로**: 실제 존재하는 경로인지 확인
- **코드 참조**: 정확한 라인 번호와 내용인지 확인
- **명령어**: 올바른 명령어인지 테스트
- **설명**: 기술적으로 정확한지 검증
- **출처**: 외부 정보의 출처 명시 및 신뢰성 확인
- 잘못된 정보 전달 방지를 위해 사실 확인 필수
